# FAQ Knowledge Base — Scraping, Loading & Search Integration

## Overview

Build a FAQ knowledge base by scraping the Frequently Asked Questions section from [marykay.com/en/contact-us.html](https://www.marykay.com/en/contact-us.html#frequently-asked-questions) and integrate it into the existing AI Search pipeline (Azure AI Search + GPT-4.1 RAG). This enables the AI search to answer common customer questions such as "How do I become a Mary Kay consultant?", "What is the return policy?", or "How do I find a consultant near me?" — grounded in official Mary Kay FAQ content.

---

## Source Data

**URL:** `https://www.marykay.com/en/contact-us.html#frequently-asked-questions`

The FAQ section is organized into **3 categories** with collapsible Q&A pairs:

| Category | Questions |
|---|---|
| **Ordering Products** | 6 questions (purchasing, finding a consultant, order inquiries, cancellation, returns/exchanges, online order contact) |
| **Shipping & Delivery** | 1 question (international delivery) |
| **Become a Beauty Consultant** | 1 question (how to become a consultant) |

**Total:** ~8 FAQ entries (subject to change as Mary Kay updates the page).

---

## Current Behavior

- AI Search only queries the `marykay-products` index, which contains product pages scraped from marykay.com.
- FAQ-style questions (e.g., "How do I return a product?", "How to become a consultant?") return **product results** that do not answer the user's actual question.
- The GPT-4.1 RAG answer is forced to speculate or say "I couldn't find relevant information" because no FAQ content exists in the index.

## Requirements

### R1: FAQ Scraper

| Item | Detail |
|---|---|
| **Target URL** | `https://www.marykay.com/en/contact-us.html` |
| **Section** | `#frequently-asked-questions` |
| **Parser** | BeautifulSoup or Playwright (if JS-rendered) |
| **Output format** | JSONL file (`faq_records.jsonl`) |

Each FAQ record should contain:

```json
{
  "id": "faq-ordering-return-policy",
  "source_url": "https://www.marykay.com/en/contact-us.html#frequently-asked-questions",
  "category": "Ordering Products",
  "question": "Can I return or exchange a Mary Kay product?",
  "answer": "Products may be replaced, exchanged, or refunded in full upon return within one year...",
  "content": "Q: Can I return or exchange a Mary Kay product?\nA: Products may be replaced, exchanged, or refunded in full upon return within one year...",
  "page_type": "faq",
  "locale": "en_US",
  "scraped_at": "2026-03-26T00:00:00Z"
}
```

- The `content` field concatenates question + answer for full-text and vector search.
- The `id` field uses a slug derived from category + question for deduplication.

### R2: Load FAQ Records into Azure AI Search

| Item | Detail |
|---|---|
| **Index** | `marykay-products` (existing) or new `marykay-faq` index |
| **Approach A (preferred)** | Add FAQ documents to the existing `marykay-products` index with `page_type: "faq"` so they are searchable alongside products in a single query. |
| **Approach B (alternative)** | Create a separate `marykay-faq` index and query both indices in the AI search pipeline, merging results before sending to GPT-4.1. |
| **Embedding** | Generate `content_vector` using the same `text-embedding-3-large` model (3072-dim) used for product records. |
| **Fields mapping** | `title` = question, `main_text` = answer, `category` = FAQ category, `url` = source URL, `page_type` = "faq" |

**Recommended: Approach A** — single index keeps the search pipeline simple and allows hybrid (keyword + vector + semantic) retrieval to naturally surface FAQ content when relevant.

### R3: Update AI Search Pipeline

Modify `ai_search.py` to handle FAQ results in the RAG context:

1. **Search results** — FAQ records will appear in `search_products()` results when they match the query. No code change needed if using Approach A.
2. **Context building** — Update the context builder in `ask()` to format FAQ results differently from product results:
   ```
   [FAQ 1]
   Category: Ordering Products
   Question: Can I return or exchange a Mary Kay product?
   Answer: Products may be replaced, exchanged, or refunded in full...
   Source: https://www.marykay.com/en/contact-us.html
   ```
3. **System prompt** — Update the GPT-4.1 system prompt to acknowledge FAQ content:
   - "You are a Mary Kay product advisor **and customer support assistant**."
   - "When FAQ content is provided, use it to answer customer questions directly."
   - "Cite the FAQ source when answering FAQ-based questions."

### R4: Update Frontend Display

- When AI search returns FAQ-type sources (`page_type: "faq"`), render them differently from product cards:
  - Show as an **info card** with the question as title and answer as body.
  - Link to the FAQ page on marykay.com.
  - Use a distinct badge (e.g., `badge-faq`) to differentiate from product/content cards.

### R5: Enrichment in `/api/ai-search` Endpoint

Update `app.py` to handle FAQ sources:

- If a source has `page_type: "faq"`, pass it through without requiring `content_hash` or local record matching.
- Set `url` to the marykay.com FAQ page URL.
- Skip image/price fields (not applicable to FAQ entries).

## Acceptance Criteria

1. FAQ scraper extracts all Q&A pairs from the contact-us page and produces valid JSONL.
2. FAQ records are indexed in Azure AI Search with proper embeddings and `page_type: "faq"`.
3. Searching "return policy" or "how to become a consultant" in AI mode returns a grounded answer citing FAQ content.
4. FAQ sources display as distinct info cards (not product cards) in the UI.
5. All FAQ links point to `https://www.marykay.com/en/contact-us.html#frequently-asked-questions`.
6. Existing product search behavior is not affected.

---

## Dependencies

- Existing Azure AI Search index (`marykay-products`) and embedding model (`text-embedding-3-large`)
- BeautifulSoup4 or Playwright for scraping
- Existing scraper virtual environment (`scraper/.venv`)
- Azure AI Search SDK (`azure-search-documents`)

---

## Out of Scope

- Scraping FAQ pages for other locales (e.g., `es_US`) — future enhancement.
- Automated periodic re-scraping (cron/scheduled) — manual re-run for now.
- Scraping other support pages beyond the FAQ section.
