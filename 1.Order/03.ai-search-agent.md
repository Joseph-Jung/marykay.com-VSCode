# AI Search Agent — Natural Language Product Search via Microsoft Foundry

## Overview

Replace the local `documents.jsonl`-based search with a cloud-hosted **Azure AI Search** index powered by **Microsoft Foundry**. Users will ask product questions in plain English and receive grounded, citation-backed answers generated by GPT-4.1 over the indexed Mary Kay product catalog.

## Implementation Plan

### Phase 1 — Create Azure AI Search Index

1. **Provision an Azure AI Search resource** (if not already available in the Foundry project).
2. **Define the index schema** mapping to the fields in `documents.jsonl`:

   | Index Field | Source Field | Type | Searchable | Filterable | Facetable |
   |---|---|---|---|---|---|
   | `id` | derived (hash of `canonical_url`) | `Edm.String` | — | yes | — |
   | `url` | `canonical_url` | `Edm.String` | — | yes | — |
   | `title` | `title` | `Edm.String` | yes | — | — |
   | `h1` | `h1` | `Edm.String` | yes | — | — |
   | `description` | `meta_description` | `Edm.String` | yes | — | — |
   | `main_text` | `main_text` | `Edm.String` | yes | — | — |
   | `product_name` | `product_fields.name` | `Edm.String` | yes | yes | yes |
   | `price` | `product_fields.price` | `Edm.String` | yes | yes | yes |
   | `size` | `product_fields.size` | `Edm.String` | yes | — | — |
   | `category` | `product_fields.category` | `Edm.String` | yes | yes | yes |
   | `key_benefits` | `product_fields.key_benefits` | `Collection(Edm.String)` | yes | — | — |
   | `ingredients` | `product_fields.ingredients` | `Collection(Edm.String)` | yes | — | — |
   | `how_to_use` | `product_fields.how_to_use` | `Edm.String` | yes | — | — |
   | `shade_options` | `product_fields.shade_options` | `Collection(Edm.String)` | yes | yes | yes |
   | `images` | `images` | `Collection(Edm.String)` | — | — | — |
   | `content_vector` | *(generated at ingestion)* | `Collection(Edm.Single)` | — | — | — |

3. **Enable semantic ranking** on the index (semantic configuration over `title`, `main_text`, `key_benefits`).
4. **Enable vector search** using an Azure OpenAI embedding model (e.g., `text-embedding-3-large`) to populate the `content_vector` field.

### Phase 2 — Data Ingestion

1. **Transform `documents.jsonl`** into the index schema format:
   - Flatten nested `product_fields` into top-level index fields.
   - Generate a stable `id` from each record's `canonical_url`.
   - Generate embedding vectors for a composite text field (`title` + `h1` + `main_text` + `key_benefits`).
2. **Upload documents** to Azure AI Search via the REST API or Python SDK (`azure-search-documents`).
3. **Validate** by running sample queries against the index and confirming result quality.

**Ingestion Script** (conceptual — Python):

```python
from azure.search.documents import SearchClient
from azure.search.documents.indexes import SearchIndexClient
from azure.core.credentials import AzureKeyCredential
import json, hashlib

SEARCH_ENDPOINT = "<azure-ai-search-endpoint>"
SEARCH_KEY = "<azure-ai-search-admin-key>"
INDEX_NAME = "marykay-products"

search_client = SearchClient(SEARCH_ENDPOINT, INDEX_NAME, AzureKeyCredential(SEARCH_KEY))

docs = []
with open("documents.jsonl") as f:
    for line in f:
        rec = json.loads(line)
        pf = rec.get("product_fields", {})
        doc = {
            "id": hashlib.sha256(rec["canonical_url"].encode()).hexdigest()[:32],
            "url": rec.get("canonical_url", ""),
            "title": rec.get("title", ""),
            "h1": rec.get("h1", ""),
            "description": rec.get("meta_description", ""),
            "main_text": rec.get("main_text", ""),
            "product_name": pf.get("name", ""),
            "price": pf.get("price", ""),
            "size": pf.get("size", ""),
            "category": pf.get("category", ""),
            "key_benefits": pf.get("key_benefits", []),
            "ingredients": pf.get("ingredients", []),
            "how_to_use": pf.get("how_to_use", ""),
            "shade_options": pf.get("shade_options", []),
            "images": [img.get("src", "") for img in rec.get("images", []) if isinstance(img, dict)],
        }
        docs.append(doc)

# Upload in batches of 100
for i in range(0, len(docs), 100):
    batch = docs[i:i+100]
    search_client.upload_documents(documents=batch)

print(f"Uploaded {len(docs)} documents to index '{INDEX_NAME}'")
```

### Phase 3 — Query Pipeline (Search + GPT-4.1 RAG)

The application receives a user's natural language query and:

1. **Searches Azure AI Search** using hybrid retrieval (keyword + vector + semantic reranking) to get the top-K most relevant product documents.
2. **Constructs a prompt** for GPT-4.1 that includes the retrieved documents as grounding context.
3. **Calls GPT-4.1** via the Foundry chat model endpoint to generate a natural language answer.
4. **Returns** the answer with product citations and links.

**Query Flow** (conceptual — Python):

```python
import requests

AIPROJECT_ENDPOINT = "https://josephjung-7921-resource.services.ai.azure.com/api/projects/josephjung-7921"
CHAT_MODEL_ENDPOINT = "https://josephjung-7921.cognitiveservices.azure.com/openai/deployments/gpt-4.1/chat/completions?api-version=2024-05-01-preview"
API_KEY = "<api-key>"

def search_products(query: str, top_k: int = 5) -> list:
    """Search Azure AI Search index for relevant products."""
    results = search_client.search(
        search_text=query,
        query_type="semantic",
        semantic_configuration_name="marykay-semantic-config",
        top=top_k,
        select=["title", "url", "main_text", "product_name", "price", "category", "key_benefits"]
    )
    return [dict(r) for r in results]

def ask(user_query: str) -> dict:
    """Natural language product search with grounded GPT-4.1 answer."""
    # Step 1: Retrieve
    search_results = search_products(user_query)
    context = "\n\n---\n\n".join(
        f"**{r['product_name'] or r['title']}** ({r['price']})\n"
        f"Category: {r['category']}\n"
        f"Benefits: {', '.join(r.get('key_benefits', []))}\n"
        f"URL: {r['url']}\n"
        f"{r['main_text'][:500]}"
        for r in search_results
    )

    # Step 2: Generate
    headers = {"Content-Type": "application/json", "api-key": API_KEY}
    body = {
        "messages": [
            {"role": "system", "content": (
                "You are a Mary Kay product advisor. Answer the user's question "
                "using ONLY the product information provided below. "
                "Cite product names and URLs. If the answer is not in the context, say so.\n\n"
                f"## Product Context\n\n{context}"
            )},
            {"role": "user", "content": user_query}
        ],
        "temperature": 0.3,
        "max_tokens": 1024
    }
    resp = requests.post(CHAT_MODEL_ENDPOINT, headers=headers, json=body)
    answer = resp.json()["choices"][0]["message"]["content"]

    return {
        "answer": answer,
        "sources": [{"title": r["title"], "url": r["url"]} for r in search_results]
    }
```

### Phase 4 — Run Both Search Modes Side by Side

The AI Search feature is **additive** — the existing local BM25/TF-IDF keyword search remains fully operational.

- **Keyword Search** (`/api/search`): Existing local search over `documents.jsonl` — unchanged, stays as default.
- **AI Search** (`/api/ai-search`): New Azure AI Search + GPT-4.1 RAG pipeline — added as a separate mode.
- The frontend provides a toggle so users can switch between modes and compare results.
- `documents.jsonl` continues to power the local keyword search at runtime and serves as the source-of-truth for re-indexing the Azure AI Search index.
- A scheduled re-crawl + re-index pipeline can keep the Azure index fresh.
