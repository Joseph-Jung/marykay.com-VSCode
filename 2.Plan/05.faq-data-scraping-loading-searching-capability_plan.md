## Execution Steps

### Step 1 — Create FAQ Scraper

**File:** `scraper/faq_scraper.py` (new)
**Requirement:** R1

Build a scraper targeting `https://www.marykay.com/en/contact-us.html` that extracts the FAQ section:

1. Fetch the page HTML using `httpx` (consistent with existing `scraper.py`).
2. Parse with BeautifulSoup to locate the FAQ section (`#frequently-asked-questions`).
3. Extract each Q&A pair with its parent category (Ordering Products, Shipping & Delivery, Become a Beauty Consultant).
4. For each FAQ entry, produce a record:

```json
{
  "id": "faq-ordering-return-policy",
  "source_url": "https://www.marykay.com/en/contact-us.html#frequently-asked-questions",
  "category": "Ordering Products",
  "question": "Can I return or exchange a Mary Kay product?",
  "answer": "Products may be replaced, exchanged, or refunded in full...",
  "content": "Q: Can I return or exchange a Mary Kay product?\nA: Products may be replaced, exchanged, or refunded...",
  "page_type": "faq",
  "locale": "en_US",
  "scraped_at": "2026-03-26T00:00:00Z"
}
```

5. Write all records to `scraper/faq_records.jsonl`.

**Verify:** Run `python scraper/faq_scraper.py` and confirm `faq_records.jsonl` contains ~8 records with valid Q&A content.

---

### Step 2 — Create FAQ Loader (Embed + Upload to Azure AI Search)

**File:** `scraper/faq_loader.py` (new)
**Requirement:** R2

Build a loader script that reads `faq_records.jsonl` and uploads FAQ documents to the existing `marykay-products` Azure AI Search index (Approach A — single index):

1. Read each FAQ record from `faq_records.jsonl`.
2. Generate `content_vector` embedding for the `content` field using the `text-embedding-3-large` endpoint (same as product embeddings).
3. Map FAQ fields to index fields:

| FAQ Field | Index Field |
|---|---|
| `id` | `id` (unique document key) |
| `question` | `title` |
| `answer` | `main_text` |
| `content` | `description` (for full-text search) |
| `category` | `category` |
| `source_url` | `url` |
| `page_type` | `page_type` |
| `locale` | `locale` |
| `content_vector` | `content_vector` (3072-dim float array) |

4. Upload documents using `SearchClient.upload_documents()` (merge-or-upload mode for idempotency).

**Note:** Before uploading, verify that the `marykay-products` index schema includes a `page_type` field. If not, add it to the index definition. Also verify the index allows documents without product-specific required fields (price, size, etc.) — FAQ documents will have empty values for those fields.

**Verify:** Run `python scraper/faq_loader.py` and confirm documents appear in Azure AI Search. Test with a manual query in the Azure portal for "return policy".

### Step 3 — Update AI Search Context Builder for FAQ Content

**File:** `search-site/ai_search.py`
**Requirement:** R3

Modify the `ask()` function to detect and format FAQ results differently in the GPT-4.1 context:

1. In the context-building loop, check if a result's `page_type` is `"faq"` (or if product-specific fields like `price` are empty while `main_text` contains Q&A patterns).
2. Format FAQ results as:

```
[FAQ 1]
Category: Ordering Products
Question: Can I return or exchange a Mary Kay product?
Answer: Products may be replaced, exchanged, or refunded in full upon return within one year...
Source: https://www.marykay.com/en/contact-us.html
```

3. Keep product results formatted as before (with Name, Price, Size, etc.).

**Verify:** Call `ai_search.ask("return policy")` and inspect the context sent to GPT-4.1 — FAQ entries should use the FAQ format.

### Step 5 — Handle FAQ Sources in `/api/ai-search` Endpoint

**File:** `search-site/app.py`
**Requirement:** R5

Update the enrichment loop in `api_ai_search()` to handle FAQ-type sources:

```python
for source in result.get("sources", []):
    if source.get("page_type") == "faq":
        # FAQ sources don't need local record matching
        source["content_hash"] = None
        source["image_url"] = ""
        source["image_alt"] = ""
        source["snippet"] = (source.get("main_text", "") or
                             source.get("description", ""))[:300]
        source["locale"] = source.get("locale", "en_US")
        enriched_sources.append(source)
        continue

    # Existing product enrichment logic...
    local_rec = _find_local_record(source.get("url", ""))
    ...
```

This ensures FAQ sources pass through without attempting `_find_local_record()` lookups (which would fail since FAQs aren't in the local `record_map`).

**Verify:** Confirm `/api/ai-search` returns FAQ sources with `page_type: "faq"` and appropriate fields.

   - No price display.

```javascript
if (r.page_type === "faq") {
    return `
      <div class="result-card faq-card" onclick="window.open('${escapeHtml(r.url)}', '_blank')">

        <div class="card-faq-icon">&#x2753;</div>
        <div class="card-title">${escapeHtml(r.product_name || r.title)}</div>
```

- No existing product data or search behavior is modified.
